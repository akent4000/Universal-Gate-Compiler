## Фаза 2: Булевский и Структурный Синтез (Максимальный ROI)
*Текущий алгебраический конвейер дошел до своего предела на сложных арифметических бенчмарках (напр. BCD-to-7seg). Для преодоления порога в ~300 гейтов требуются структурные эвристики "взрослых" САПР.*

* [*] **Многовыходовая минимизация (Multi-output Espresso):** *Готово. Внедрен в `pipeline.py`, общие кубы извлекаются।*
* [*] **Алгебраическое деление (Subexpression Extraction):** *Готово. `multi_output_factorize` извлекает базовые общие алгебраические делители।*
* [*] **Переход на AIG (And-Inverter Graph):** *Готово. `aig.py` обеспечивает быстрое структурное хеширование.*
* [*] **Локальный AIG-рерайтинг (DAG-aware Rewriting):** **(КРИТИЧЕСКИЙ ПРИОРИТЕТ)** Реализовать алгоритм (аналог команды `rewrite` в Berkeley ABC), который многократно обходит AIG, вырезает подграфы из 4-6 узлов и заменяет их на минимальные прекомпилированные шаблоны (Boolean Matching). Это решит проблему раздувания логики на арифметических операциях (XOR-структурах).
* [ ] **Функциональная декомпозиция (Ashenhurst-Curtis / Roth-Karp):** **(ВЫСОКИЙ ПРИОРИТЕТ)** Разработать алгоритм поиска независимых подфункций $f(X) = g(h(X_{bound}), X_{free})$. Это позволит алгоритму самому находить "узкие места" (например, 8-битный BCD-код внутри 7-битного бинарного входа), радикально уменьшая общую площадь схемы без ручного разбиения `.pla` файлов.
* [ ] **Exact Synthesis (SAT-based):** Опционально: использовать SAT-солвер для поиска гарантированно минимального количества NAND-гейтов для небольших окон логики (до 6 переменных).
