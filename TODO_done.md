## Фаза 1: Инфраструктура, Бенчмарки и Надежность

* [x] **Расширение базы MCNC-бенчмарков:** * Внедрить индустриальные стресс-тесты для отслеживания регрессий качества: `RD53` (popcount), `Parity` (9-bit XOR-деревья), умножители `3x3` и `4x4`, а также плотные автоматы `misex1` и `z4ml`.
  * Эти тесты покажут реальную эффективность синтезатора на различных типах логики (арифметика vs. управление).
* [x] **Формальная верификация (Miter-схемы через SAT):** * Полный перебор (Exhaustive simulation) не работает для схем $>20$ входов. Необходимо написать функцию формальной проверки эквивалентности через `z3-solver`.
  * **Как это работает:** Строится "miter" (схема-сравнение), где выходы исходной таблицы истинности и сгенерированной NAND-сети попарно соединяются через `XOR`, а их результаты объединяются через `OR`. Если SAT-солвер выдает `UNSAT` (невозможно получить 1 на выходе), схемы гарантированно идентичны.
* [x] **Property-Based Testing (Случайные графы):** * Использовать библиотеки (например, `Hypothesis` для Python) для генерации случайных мутаций таблиц истинности и фрагментов AIG с автоматической проверкой их эквивалентности после агрессивных оптимизаций.
* [x] **Профилирование производительности (Profiling):** * Внедрить замеры времени исполнения и потребления памяти для каждого прохода (Espresso, Factorization, Rewriting). Это поможет выявлять экспоненциальные взрывы алгоритмов на ранних этапах.

## Фаза 2: Булевский и Структурный Синтез

* [x] **Многовыходовая минимизация (Multi-output Espresso):** Общие кубы успешно извлекаются.
* [x] **Извлечение ядер и ко-ядер (Kernels and Co-kernels):** Реализован классический алгоритм факторизации Бреятона (`brayton_factor` в `optimize.py`) — перечисление ядер по Brayton-McMullen, алгебраическое деление, выбор лучшего ядра по выигрышу литералов и рекурсивная факторизация. Пример `a*c + a*d + b*c + b*d` сжимается в `(a + b) * (c + d)` (8 → 4 литералов).
* [x] **Функциональная декомпозиция (Ashenhurst-Curtis / Roth-Karp):** Реализован алгоритм поиска независимых подфункций $f(X) = g(h_1(X_{bound}), \ldots, h_k(X_{bound}), X_{free})$ в [nand_optimizer/decomposition.py](nand_optimizer/decomposition.py) — перебор бипартиций входов, тест множественности столбцов декомпозиционной диаграммы Рота-Карпа ($\mu \le 2^k$), жадная кластеризация строк с don't-cares, синтез $h_i$ и $g$ через Espresso, выбор по литерному выигрышу. На RD53-popcount даёт 80 → 24 литерала для LSB.
  * **Цель:** Позволить алгоритму автоматически находить логические "бутылочные горлышки" внутри плоских таблиц истинности (например, выявить, что схема вычисляет промежуточный 8-битный код перед финальным ответом) и разбивать задачу на две независимые, радикально уменьшая общую площадь.
* [x] **Exact Synthesis (SAT-based Boolean Matching):** Реализован SAT-солвер (Z3) в [nand_optimizer/exact_synthesis.py](nand_optimizer/exact_synthesis.py) — параллельный per-minterm Boolean-энкодинг с one-hot селекторами портов и симметрийным правилом `l_idx < r_idx`, итеративный поиск по возрастанию числа AND-гейтов. Результаты кешируются в памяти и сразу используются рерайтером (`rewrite_aig(..., use_exact=True)`). На проверках: XOR₂ → 3 AND, MAJ₃ → 4 AND, 3-XOR → 6 AND, все с верификацией через `evaluate_template`.

### Автоматическая иерархическая декомпозиция (bin→BCD→7seg gap)

* [x] **(1) Рекурсивная Ashenhurst-Curtis:** Реализовано в [`ashenhurst_decompose_recursive`](nand_optimizer/decomposition.py) + [`_decompose_rec`](nand_optimizer/decomposition.py). Рекурсивно разбивает `f = g(h_1..h_k, X_free)`, останавливается при `|support| ≤ 4` (порог NPN-DB) или `depth ≥ 4`, whole-tree bake-off через snapshot/restore. Результат: mult3 74→73 NAND, mult4 418→417, excess3/7seg/adder без регрессий.
* [x] **(2) Shared-support декомпозиция для группы выходов:** Реализовано в [`multi_output_decompose`](nand_optimizer/decomposition.py) + вызывается в [`pipeline.py`](nand_optimizer/pipeline.py) на шаге [3.5] перед per-output _phase2. Строит объединённую decomposition chart по всем выходам, ищет единый h-bus через joint column multiplicity. Bake-off на shared AIG vs per-output build; applied на rd53. Интеграция `SharedDecompositionResult` → `shared_bindings` → `_phase2(shared_prebuilt=...)`. Ключевой пункт для bin→BCD→7seg gap (применяется автоматически когда μ ≤ 2^k при n_inputs ∈ [3,12], n_outputs ≥ 2).
* [x] **(3) Bi-decomposition (OR/AND/XOR) в rewrite:** Реализовано в [`nand_optimizer/bidec.py`](nand_optimizer/bidec.py) (`bidec_aig`) и зарегистрировано как команда `bidec` в [`script.py`](nand_optimizer/script.py). Для каждого AND-узла с k-cut в диапазоне `[min_cut, cut_size]` (по умолчанию 5–8) функция [`bi_decompose_tt`](nand_optimizer/bidec.py) ищет разбиение `f = g(X) op h(Y)` для всех дизъюнктных `(X, Y)` при `|X|,|Y| ≤ 4` (или ≤ 6 с `-z`). AND-тест использует `∃`-квантификацию по cofactor'ам, OR-тест — `∀`-квантификацию, XOR-тест — проверку `f(x,y) ⊕ f(x,0) ⊕ f(0,y) ⊕ f(0,0) == 0` для всех `(x,y)`. Каждая половина синтезируется через `AIG_DB_4` (с корректным 4-input TT-expansion и шринком шаблона, см. `_expand_tt_to_4`/`_shrink_template`); комбинирование через `make_and`/`make_or`/`make_xor`. Cost-model идентичный `rewrite_aig`: принимаем только при `n_new - mffc_size < base_cost`. Использование: `bidec [-K N] [-k N] [-r N] [-z]`.
* [x] **(4) BDD-based декомпозиция через «узкие сечения»:** Реализовано в [`nand_optimizer/bdd_decomp.py`](nand_optimizer/bdd_decomp.py) (`bdd_decompose_aig`) и зарегистрировано как команда `bdd`. Для каждого выхода с `|support| ≤ max_inputs` (по умолчанию 16) строится ROBDD через рекурсивный Shannon split над таблицей истинности выхода, запускается `bdd.reorder()` (sifting) для минимизации размера, далее BDD канонически реализуется в целевой AIG через ITE-композицию (`_realize_bdd_in_aig` корректно обрабатывает complement-edges dd.autoref через флаг `.negated`). Bake-off snapshot/restore: оставляем BDD-форму только если `added < cone_size` (строго меньше исходного конуса). Общие под-BDD-узлы между выходами сливаются автоматически через structural hashing в `new_aig`. Использование: `bdd [-K max_inputs]`. No-op если пакет `dd` не установлен.
* [x] **(5) SAT-based resubstitution >4 входов:** Реализовано в [`nand_optimizer/sat_resub.py`](nand_optimizer/sat_resub.py) (`resub_aig`) и зарегистрировано как команда `resub`. Для каждого AND-узла `v` с k-cut в `[cut_size_min, cut_size_max]` (по умолчанию 5–7) функция [`_simulate_over_cut`](nand_optimizer/sat_resub.py) одним topological sweep'ом вычисляет 2^k-бит TT для каждого AND-узла, чей support ⊆ cut; [`_select_divisors`](nand_optimizer/sat_resub.py) отбирает до `max_divisors` кандидатов (cut-vars + AND-узлы TFI вне MFFC(v)); для `m ∈ {1..3}` и каждой m-ки дивизоров [`_functional_dependency`](nand_optimizer/sat_resub.py) проверяет: существует ли `F: {0,1}^m → {0,1}` такая, что `v(cut) ≡ F(d_1(cut)..d_m(cut))`? — точный функциональный тест над всеми 2^k минтермами куба. Если зависимость есть, F задаётся уникально; синтезируется через `AIG_DB_4`. Принимаем при `n_new - |MFFC(v)| < base_cost`. Соответствует `dc2`-resub в ABC для узких кубов, но работает поверх **exact TT over the cut**. Использование: `resub [-K N] [-k N] [-M N] [-D N] [-r N]`.

**Замечание:** Все три прохода (bidec/bdd/resub) используют одинаковую cost-model (`n_new - mffc_size < base_cost`) и совместимы с существующими командами script'а (`balance; rewrite; fraig; dc; bidec; resub; bdd; balance`). Вспомогательные функции `_expand_tt_to_4` + `_shrink_template` обеспечивают корректный lookup в `AIG_DB_4` для m<4-input функций (латентный баг в `rewrite.py` для `k<4` здесь устранён явно).

* [x] **Hierarchical / multi-stage synthesis (`--compose`, `hierarchical_optimize`):** Реализовано в [`pipeline.py::hierarchical_optimize`](nand_optimizer/pipeline.py). Пользователь описывает цепочку стадий в JSON-spec'е: каждая стадия — своя `.pla` плюс `connect` map, привязывающий её входы к именованным выходам предыдущих стадий (опциональный `rename` переименовывает выходы). Алгоритм: (1) каждая уникальная `TruthTable` оптимизируется независимо через `rewrite; fraig; balance; rewrite -z; fraig; balance`; (2) стадии объединяются в один shared AIG через `AIG.compose()` с substitution shared-сигналов — structural hashing автоматически сливает общие подвыражения между стадиями; (3) общий AIG GC'ится и прогоняется через post-script; (4) NAND-маппинг даёт единый `OptimizeResult`. CLI: `--compose path/to/spec.json`, пример spec'а — [myPLAfiles/bcd_7seg_composition.json](myPLAfiles/bcd_7seg_composition.json).
* [x] **Auto-compose: автоматическая детекция симметричных output-групп:** Модуль [`nand_optimizer/auto_compose.py`](nand_optimizer/auto_compose.py). Для данной плоской `TruthTable` ищет равнопарные группы выходов, реализующие *одну и ту же* комбинационную функцию через разные intermediate сигналы (типичный паттерн — `Binary_to_7seg_0-99`: десятки и единицы разделяют логику `BCD→7seg`). Детектор проверяет, что оба множества output-column-паттернов совпадают на всех care-минтермах; присваивает `k = ⌈log₂ |patterns|⌉` бит промежуточной шине; генерирует две `.pla`: `<stem>_intermediate.pla` (n_inputs → 2k выходов) и `<stem>_decoder.pla` (k входов → group_width выходов); записывает JSON spec. CLI: `python -m nand_optimizer circuit.pla --auto-compose` — делает детекцию, пишет промежуточные PLA рядом с исходной, и сразу запускает `hierarchical_optimize` поверх них. Fallback на flat-синтез, если симметрия не обнаружена. Мотивация: плоский синтез `Binary_to_7seg_0-99` — 370 NAND, ручная композиция `Binary_to_BCD + 2×Binary_to_7seg` — 215 NAND.

## Фаза 2.5: Технический долг и исправление маппинга

* [x] **Учет Global Fanout в AIG Rewriter (Причина раздувания №1):** Полностью переписан [nand_optimizer/rewrite.py](nand_optimizer/rewrite.py) с эвристикой, осознающей fanout. Подсчитываются ссылки (`_compute_ref_counts`) по всему AIG включая выходы; для каждого кандидата-cut вычисляется MFFC (Maximum Fanout-Free Cone) через стандартный kill-ref алгоритм, а стоимость нового шаблона считается non-destructive через виртуальные литералы (`_count_template_new_nodes`) с учётом intra-template sharing и структурного хеша нового AIG. Замена допускается только при строгом `n_new < mffc_size`. На бенчмарках без регрессий и с реальными выигрышами: misex1 75→58 (−23%), rd53 51→47 (−4), adder 20→19; 7seg/excess3/z4ml/parity9/mult3 без изменений; miter-верификация подтверждает эквивалентность. Пайплайн активирован (`rounds=1`).
* [x] **Продвинутый AIG-to-NAND маппинг (Bubble Pushing / Причина №2):** Реализован pre-pass `_bubble_push` в `aig_to_gates` ([nand_optimizer/nand.py](nand_optimizer/nand.py)). Алгоритм ищет паттерн `AND(pos_v, c)` где `pos_v` — позитивный полярный вход (требует инвертора). Если в AIG уже есть узел `m = AND(a, c)` или `AND(b, c)` (где `a,b` — входы `v`) с `pos_m` уже необходимым другому потребителю — реструктурирует узел на `AND(b, pos_m)`, устраняя отдельный инвертор для `pos_v`. Итеративный до фиксированной точки (≤8 раундов), с топологической проверкой безопасности. Результат на Binary_to_7seg: **381 → 373 гейтов** (−8), инверторы **105 → 100** (−5). Все тесты, верификация z3 и бенчмарки MCNC проходят без регрессий.
* [x] **Очистка фантомных узлов (AIG Garbage Collection):** Реализован метод `AIG.gc(out_lits)` ([nand_optimizer/aig.py](nand_optimizer/aig.py)): DFS от выходных литералов помечает достижимые узлы, затем перестраивает компактный AIG только из них — обе структуры `_nodes` и `_hash` содержат исключительно живые узлы. `rewrite_aig` вызывает `gc` в конце каждого раунда ([nand_optimizer/rewrite.py](nand_optimizer/rewrite.py)), устраняя мёртвые трансляции MFFC-потомков. Побочный эффект: структурный хеш в следующем раунде не засорён фантомными записями, поэтому cost-model `_count_template_new_nodes` точнее оценивает коллизии.
* [x] **Прямая генерация AIG (Устранение AST-оверхеда):** * Убрать промежуточные конвертации логики: `AST -> Legacy Builder -> AIG`. Строить AIG сразу после минимизации (или парсинга). Логические оптимизации (например, разложение Шеннона) выполнять исключительно как быстрые перекидывания указателей на графе.
* [x] **Извлечение XOR/XNOR-структур (Boolean Division):** * Алгоритмы типа Espresso сплющивают всю логику в сумму произведений (SOP). Для функций чётности (XOR) это приводит к экспоненциальному взрыву числа литералов.
  * **Решение:** Реализован проход `_detect_xor_patterns` в [nand_optimizer/nand.py](nand_optimizer/nand.py). Сканирует все AND-узлы в AIG на наличие паттерна `n3=AND(~n1,~n2)` где `n1=AND(p,q)`, `n2=AND(p^1,q^1)`. Проверяет эксклюзивность: `n1` и `n2` используются только `n3` (fanout=1, positive polarity не нужна). Для найденных паттернов в `aig_to_gates` вместо стандартной 5-NAND формы (с `NOT(x)` и `NOT(y)`) эмитируется канонический 4-NAND XOR: `g1=NAND(p,q^1)`, `g2=NAND(p,g1)`, `g3=NAND(q^1,g1)`, `out=NAND(g2,g3)`. На parity9 извлекается 4 XOR-паттерна; верификация через z3 подтверждает эквивалентность на всех MCNC-бенчмарках.

## Фаза 2.7: Продвинутые манипуляции с графами

* [x] **FRAIGing (Functionally Reduced AIGs):** Реализован в [nand_optimizer/fraig.py](nand_optimizer/fraig.py) (Mishchenko et al., 2005). Пайплайн: (1) bit-parallel псевдослучайная симуляция всех узлов AIG с W-битной сигнатурой, (2) разбиение узлов на классы по сигнатуре (включая классы «равен константе»), (3) для каждой пары-кандидата Z3-miter доказывает эквивалентность (UNSAT ⇒ merge), при SAT — контрпример добавляется в паттерны и классы пересчитываются, (4) реконструкция компактного AIG с заменой слитых узлов и `gc`. Интегрирован в [pipeline.py](nand_optimizer/pipeline.py) между rewrite и balance; убирает скрытую избыточность, недоступную алгебраическим проходам.
* [x] **AIG Balancing (Балансировка графа):** * Написать алгоритм реструктуризации графа для минимизации его логической глубины (сокращение числа последовательных узлов) с сохранением площади. Критически важно для оптимизации задержек распространения сигнала (Timing). Реализован в [nand_optimizer/balance.py](nand_optimizer/balance.py): сбор листьев AND-дерева с поглощением однофанаутных узлов + min-heap для построения минимально-глубокого дерева. Площадь сохраняется: n листьев → n−1 AND. Интегрирован как проход [8] в `pipeline.py`, параметр `balance=True`.

Для cut'ов $k=5,6$ экзакт-синтез кеширует результаты в памяти и подключается по требованию.
* [x] **Don't-Care оптимизация — V1 (SDC):** Реализован DC-aware rewrite-проход в [nand_optimizer/dont_care.py](nand_optimizer/dont_care.py). Для каждой k-feasible cut узла v: (1) bit-parallel sim + Z3 SAT доказывают satisfiability-DC паттерны — cut-присваивания, недостижимые ни из одного primary-input; (2) DC-aware NPN-lookup (`_dc_aware_db_lookup`): паддинг cared-множества в 16-битное пространство через `_pad_care_to_16`, перебор DC-битов, выбор min-op шаблона из `AIG_DB_4`, согласованного с care-value на cared-позициях; (3) замена применяется только при `n_new < mffc_size` (та же cost-model, что в `rewrite_aig`); (4) финальный safety-net miter через Z3 между входным и выходным AIG — любая несоответствующая замена откатывает весь проход. Команда `dc [-K N] [-T N] [--no-sdc] [--odc]` добавлена в синтез-скрипты ([nand_optimizer/script.py](nand_optimizer/script.py)); default-script теперь `"rewrite; fraig; dc; rewrite; balance"` с post-DC rewrite-проходом, подбирающим экспонированные SDC-редукции. Интегрировано в [pipeline.py](nand_optimizer/pipeline.py) как фазы [7.7] и [7.8]. ODC-ветка V1 (windowed observability через TFO-cone + Z3 substitution miter) была unsound на арифметике (mult4 `a=14,b=3`, misex1) — отложена в V2.
* [x] **Don't-Care оптимизация — V2 (полный Mishchenko 2009):** Полная замена unsound ODC-ветки V1 на sim-based global ODC propagation, плюс три ортогональных улучшения. Всё в [nand_optimizer/dont_care.py](nand_optimizer/dont_care.py); публичный API сохранён.
  * **V2.a — Soundness fix (Mishchenko-style ODC):** Один-раз-за-вызов bit-parallel симуляция `sig_old = _simulate(aig, ..., W)` + обратно-топологическая care-propagation (`_propagate_care_sim`): для AND-узла `y = AND(a, b)` с сигнатурами $s_a, s_b$ обновление `care[a_nid] |= care[y_nid] & s_b`, симметрично для `b`. Проекция care на cut через `_project_cut_care` даёт `(care_mask, care_value)` для DC-aware lookup. Admissibility check до принятия шаблона: `_template_signature` эвалуирует AIG_DB_4-шаблон на `sig_new[cut]` и требует `(cand_sig ^ sig_old[old_id]) & care_sim[old_id] == 0` — любое sim-witnessed расхождение на cared-битах отвергает замену. `sig_new` инкрементально обновляется per-AND через `_and_signature` (O(1) как в [fraig.py:36-41](nand_optimizer/fraig.py#L36-L41)). Добавлен `_reconstruct_from_old` — рекурсивный rebuild оригинального DAG из исправных ancestors для fallthrough-случаев, когда наивный `make_and(new_a, new_b)` не проходит admissibility.
  * **V2.d — Exact synthesis с DC-mask:** Флаг `dc --dc-exact` в [script.py](nand_optimizer/script.py). Для `k > 4` cuts (где `AIG_DB_4` пуст) fall-back на [exact_synthesis.py](nand_optimizer/exact_synthesis.py) с `dc_mask = full & ~care_mask` — SAT ищет решение только на cared-позициях. Tight budget (`max_gates=5`, `timeout_ms=2000`). Admissibility-check V2.a перепроверяет результат — защита от encoder-bugs.
  * **V2.c — Window resubstitution (primary rewrite path):** Флаг `dc --no-resub` отключает. Перед AIG_DB_4 lookup сканируем существующие `new_aig` сигналы как drop-in кандидаты: `_scan_resub_0gate` (0-gate: signal совпадает с target на care-bits — экономия `mffc_size` гейтов), `_scan_resub_1gate` (пара `(u, v)` через AND с поляризациями — экономия `mffc_size - 1`). Окно по умолчанию 64 последних сигналов (`-W N`). Резуб-кандидат конкурирует с template через MFFC cost: `best_net < 0` → применяется. На mult4 `-r 3`: 27 zero-gate + 143 one-gate резубов.
  * **V2.b — Multi-round iterative refinement:** Флаг `dc -r N` (default 1). Wrapper рекурсивно вызывает `dc_optimize(..., rounds=1)` до `rounds` раз, early-exit при `n_nodes >= prev_nodes`. * [x] **Structural choice nodes — ABC `compress2rs`-style (ROADMAP P3#9):** Мультивариантный AIG с choice-цепочками между функционально эквивалентными узлами из разных структурных вариантов. Снимает фундаментальное ограничение local destructive rewrite: каждый пассаж в `rewrite; fraig; rewrite; balance` раньше уничтожал структуру предыдущего. Референс — Mishchenko, Chatterjee, Brayton («DAG-aware AIG rewriting», DAC 2006; «Integrating Logic Synthesis, Technology Mapping, and Retiming», ICCAD 2006).

  * **Core — [core/aig.py](nand_optimizer/core/aig.py):** добавлено поле `AIG._choice_next: Dict[int, int]` — singly-linked списки над node IDs; все члены одной цепочки вычисляют одну и ту же булеву функцию (same polarity, чтобы consumer-rewriter мог трактовать сиблинга как прямую альтернативу). Accessors: `add_choice(rep, alt)` (идемпотентен, O(k) по длине цепочки), `choice_class(nid)` (итератор вперёд от узла), `choice_rep(nid)` (линейный скан — длины цепочек ≤ 4), `n_choice_links()`. `gc()` расширен: при mark-phase любой живой узел затягивает в reachable всех своих choice-сиблингов (closure-итерация), чтобы rewriter мог добраться до альтернативных структур через цепочку; цепочки перестраиваются на новой нумерации с пропуском узлов, коллапснувших в один под структурным хешем. `compose()` переносит `_choice_next` из другого AIG через `lit_map`, пропуская цепочки с коллапсами. `restore(snap)` сбрасывает цепочки, ссылающиеся на роллбэкнутые node IDs.

  * **Proc `choice` — [synthesis/choice.py](nand_optimizer/synthesis/choice.py), новый модуль:** `build_choices(aig, out_lits, scripts, n_sim_patterns=256, verify_timeout_ms=2000)` запускает N variant-скриптов (по умолчанию `DEFAULT_CHOICE_SCRIPTS = ["", "balance", "rewrite", "rewrite; fraig"]`) на deep-copies исходного AIG, собирает все варианты в один комбайн через `AIG.compose()` (inputs идентифицируются по имени, общая структура автоматически сливается через structural hash). Далее: (1) bit-parallel симуляция всей комбайн-AIG, (2) бакетизация узлов по raw signature, (3) **Z3-верификация каждой кандидат-пары** через переиспользование `_build_z3_exprs` + `_check_pair` из [fraig.py](nand_optimizer/synthesis/fraig.py) (UNSAT ⇒ equiv), (4) `add_choice(rep, alt)` только для верифицированных пар **из разных вариантов** (same-variant пары уже слиты через structural hash — повторный link раздувал бы цепочки без выигрыша). Primary outputs — из варианта с наименьшим числом достижимых gate-узлов; остальные варианты остаются в комбайне как альтернативные структуры, доступные rewriter'у через цепочки. `gc()` на выходе обрезает мёртвые подграфы, но сохраняет все choice-членства живых узлов.

  * **Soundness fix — simulation-only linking отвергнут.** Первая версия `_link_choice_classes` использовала только bit-parallel симуляцию (256 паттернов). На EPFL `router.aig` это дало 42 link'а, из которых **7 оказались functionally different** при Z3-проверке — классические sim false-positives на функциях с сильно-несбалансированной TT. Итоговый rewrite-choice-aware на router'е получал неэквивалентный результат (SAT на miter'е). Решение: каждая кандидат-пара в `_link_choice_classes` теперь идёт через `_check_pair` (Z3 миттер с `verify_timeout_ms` default 2000ms), link создаётся только при `'equiv'` исходе — `'different'` / `'timeout'` оставляют узлы несвязанными. Без Z3 (import z3 fails) пасс прозрачно возвращает 0 link'ов, rewriter тогда просто работает как single-variant. Regression-тест `test_build_choices_links_are_equivalences` re-verifies каждый link через независимый Z3-вызов, защищая от регрессии в этом месте.

  * **Rewrite `-c` — [synthesis/rewrite.py](nand_optimizer/synthesis/rewrite.py):** `rewrite_aig(..., use_choices=False)` — backward-compat default; при `use_choices=True` главный цикл для каждого `old_id` строит список roots = `_choice_siblings(current_aig, old_id)` и перебирает cuts из `cuts[root]` для каждого root. Фильтр: для `root != old_id` пропускаются cuts, содержащие leaf-node `c >= old_id` (ещё не переведённый в `new_aig` через `lit_map`, плюс страхует от циклической зависимости через структурное sharing). TT оценивается на `current_aig` через `evaluate_cut_tt(current_aig, root, cut)` — root и old_id имеют одинаковую функцию по инварианту цепочки, так что template, матчащий TT, корректно воспроизводит функцию old_id через leaf-литералы, уже переведённые в new_aig. MFFC-cost считается от old_id (мы замещаем именно old_id, не root), комбинатор `best_net` сравнивает все candidate (root, cut) на равных.

  * **Script integration — [script.py](nand_optimizer/script.py):** (a) новый proc `choice [-D N] [-s STR]` в `run_script`, где `-D` — simulation word width, `-s` — pipe-separated variant-скрипты (например `"|balance|rewrite|rewrite,fraig"`, `,` внутри варианта заменяет `;`, чтобы внешний скрипт-парсер, splittящий на `;`, не разрывал variant). (b) `rewrite`/`refactor` получили флаг `-c` (`use_choices=True`). (c) `_fmt_flags` дисплеит `-c` при verbose-режиме. Списки команд и help-тексты в docstring обновлены; парсер валидирует `choice` как полноправную команду.

  * **Регрессия — [tests/test_choice_nodes.py](tests/test_choice_nodes.py), 5 инвариантов:**
    1. `test_choice_primitives_roundtrip` — `add_choice` / `choice_class` / `choice_rep` / `n_choice_links` работают корректно, включая идемпотентность, цепочки из 3 узлов и сохранение chain через `gc()`.
    2. `test_build_choices_preserves_function` — combined AIG функционально эквивалентен исходному (Z3-miter UNSAT) на OR-of-ANDs фикстуре.

    4. `test_rewrite_with_choices_preserves_function` — `rewrite_aig(use_choices=True)` на комбайн-AIG не нарушает функциональную эквивалентность.

  * **QoR (sound режим).** На 11-схемном EPFL подмножестве выигрыши скромные (0.3–0.9% поверх single-pass rewrite в sound-режиме, на `router` 230→228 ANDs, на `cavlc` 693→690, на `i2c` 1313→1310); на `adder` 0 choice links — все 4 варианта совпали. ## Фаза 3: Синтез последовательностной логики (FSM / RTL)

* [x] **Структура `StateTable` (STT):** Реализована в [nand_optimizer/fsm.py](nand_optimizer/fsm.py) — классы `StateTable` и `Transition`. Поддерживает Mealy/Moore, reset-состояние, символические входы/выходы, тернарные input-cube и output-cube (с `DASH` для don't-care), частичные спецификации (`dst=None` — next-state DC; отсутствующие (state, input) — глобальный DC). Парсер KISS2 формата (`parse_kiss()`) импортирует стандартные .kiss2-файлы.
* [x] **Примитивы памяти:** Добавлен `export_fsm_circ()` в [nand_optimizer/circ_export.py](nand_optimizer/circ_export.py) — генерирует двухуровневый .circ: sub-circuit с комбинационным конусом (NAND-only) + главный circuit с триггерами (Logisim Memory lib, компоненты "D Flip-Flop" / "J-K Flip-Flop"), часами (Clock) и обратными связями `D_i → DFF → Q_i` либо `(J_i, K_i) → JK-FF → Q_i`.
* [x] **JK-триггеры как альтернативный примитив памяти:** Параметр `excitation={'d'|'jk'}` в [`synthesize_fsm()`](nand_optimizer/fsm.py) + `flipflop={'d'|'jk'|'auto'}` в [`export_fsm_circ()`](nand_optimizer/circ_export.py). В JK-режиме [`fsm_to_truth_table`](nand_optimizer/fsm.py) эмитит по два возбуждающих бита на state bit: `J_i = K_i = T_i = Q_i XOR Q_i(t+1)` (T-fill концретизация per-bit don't-care из обратной таблицы возбуждения JK). Поскольку функции J_i и K_i совпадают, structural-hashing AIG сливает их в один общий cone автоматически — лишних гейтов нет. `simulate_fsm()` замыкает обратную связь по классической рекурренции `Q(t+1) = J·~Q + ~K·Q`, формальная эквивалентность сохраняется. Эффект на счётчиках: mod4 — 6 → 0 NAND; на последовательностных детекторах JK проигрывает (seq101: 5 → 9, partial: 2 → 7), что корректно: JK оптимален для toggle-heavy машин. CLI: `--excitation {d,jk}`.
* [x] **Алгоритмы кодирования состояний (State Encoding):** Функция `encode_states(stt, strategy)` в [nand_optimizer/fsm.py](nand_optimizer/fsm.py) — стратегии `'binary'` (`ceil(log2 n)` бит), `'onehot'` (n бит), `'gray'` (`ceil(log2 n)` бит через отражённый код Грея). Reset-состояние всегда получает all-zero код в binary/gray, чтобы включение питания (FF=0) сразу попадало в reset без дополнительной логики. CLI-флаг `--encoding {binary|onehot|gray}`.
* [x] **Генерация функций возбуждения (Excitation Logic):** Функция `fsm_to_truth_table(stt, enc)` в [nand_optimizer/fsm.py](nand_optimizer/fsm.py) проецирует STT в единую `TruthTable` с входами `Q_i ++ fsm_inputs` и выходами `D_i ++ fsm_outputs`. Per-output DC: входы unused-encoding и DASH-позиции (next-state=None или DASH в output cube) превращаются в don't-care для *каждого* выходного бита независимо — это даёт максимум свободы существующему Espresso/фактиризации. Далее `synthesize_fsm()` подаёт таблицу в обычный `optimize()` и оборачивает результат в `FSMResult`.
* [x] **Разрыв комбинаторных циклов (Feedback Loop Handling):** Комбинационный конус ацикличен *по построению*: state-биты (Q) входят как primary inputs, next-state-биты (D) выходят как primary outputs. Обратная связь `D→FF→Q` замыкается только в `export_fsm_circ` (внешний Logisim-circuit) и в `simulate_fsm()`, который шагает потактно: копирует текущее `state_bits` в входные wires, вычисляет комбинаторный граф, затем записывает новые D-биты в state на следующий такт. Ни один AIG-проход (rewrite, FRAIG, balance, miter-верификация) не видит цикл — топологическая сортировка работает в лоб.
* [x] **Минимизация состояний (Hopcroft partition refinement):** Функция `_minimize_completely_specified()` в [nand_optimizer/fsm.py](nand_optimizer/fsm.py) — классическое partition refinement: (1) стартовое разбиение по вектору выходов по всем входным паттернам, (2) итеративное дробление по сигнатуре `(successor-class-id)` для каждого входа до фиксированной точки. Перед минимизацией удаляются недостижимые от reset состояния (`_remove_unreachable`). Проверено: `redundant_detector` с 5 состояниями (3 канонических + 2 клона) корректно коллапсирует к 3 состояниям, синтезированная схема проходит 32-тактный cross-check.
* [x] **Неполностью специфицированные автоматы (Incompletely-Specified FSM):** Функция `_minimize_incompletely_specified()` в [nand_optimizer/fsm.py](nand_optimizer/fsm.py) — отношение совместимости + heuristic cover. (1) Вычисляются все output-compatible пары состояний с учётом DC в выходах; (2) для каждой пары строится множество implied-пар (преемники на каждом входе); (3) фиксируется замыкание: пара выбраковывается, если хотя бы одна implied-пара не совместима (классический метод таблиц implications); (4) Bron-Kerbosch перечисляет maximal-compatible cliques в графе совместимости; (5) greedy set-cover выбирает классы, покрывающие все состояния. Перед этим также удаляются недостижимые состояния. Проверено: `partial_detector` с частичной спецификацией коллапсирует до 3 состояний и синтезируется в 2 NAND + 2 D-FF.
* [x] **Bounded Model Checking для синтезированных FSM (`bmc_verify`):** Реализовано в [nand_optimizer/verify.py](nand_optimizer/verify.py) как `bmc_verify(fsm_result, bound=K)`. Unrolls комбинационный конус на K тактов симболически через Z3: на каждом time-frame t свободные Z3 Bool'ы кодируют FSM-входы, state-биты продвигаются вдоль референсной TT-SOP траектории, miter XOR срабатывает, если синтезированный NAND-конус расходится с референсом на любом specified выходе или next-state бите. DC-позиции (отсутствующие переходы IS-FSM, DASH в output cube) исключаются из miter'а — нет false alarms. UNSAT доказывает отсутствие расхождения для любой входной последовательности длины ≤ K; SAT-свидетельство возвращает `{step, inputs: [...], states: [...]}` — точку первого расхождения. CLI: `--bmc-bound K`, типичные значения 10–20. Закрывает разрыв между combinational miter'ом (`miter_verify`) и полной формальной верификацией автоматов.

* [x] **Асинхронный reset/preset как отдельный control-tract:** `StateTable` расширен полями `reset_input_name: Optional[str]` и `reset_polarity: {'sync'|'async_low'|'async_high'}` (default `'sync'` сохраняет прежнее поведение) в [nand_optimizer/fsm.py](nand_optimizer/fsm.py); конструктор валидирует значения и запрещает пересечение `reset_input_name` с `input_names` — reset-сигнал намеренно не участвует в excitation-логике. Поля сохраняются через `_minimize_completely_specified` / `_minimize_incompletely_specified` / `_remove_unreachable` и прокидываются в `FSMResult`. `simulate_fsm(fsm_result, input_seq, reset_seq=None)` трактует `reset_seq` как level-sensitive override: на любом цикле с активным уровнем state-биты принудительно подменяются на `reset_code` *до* эвалуации комбинационного конуса, а CLK-edge игнорируется — datasheet-поведение CLR-пина. `export_fsm_circ()` в [nand_optimizer/circ_export.py](nand_optimizer/circ_export.py) для async-режима добавляет отдельный top-level Pin + shared-tunnel `__CLR__`, раздаваемый на CLR-пин каждого D/JK-триггера мимо комбинационного конуса; для `async_low` в цепочку вставляется NAND-tied-inputs инвертор (Logisim Memory lib трактует CLR как active-high). Добавлен пример `mod4_counter_async_reset()` в [examples/fsm_examples.py](nand_optimizer/examples/fsm_examples.py) (ключ `fsm:mod4_rst`); проверено, что удержание `RESET_N=0` два такта подряд мгновенно сбрасывает счётчик и CLK игнорируется, а после снятия reset счёт корректно возобновляется с `Q0`. Закрывает требование ЛР: «при RESET=0 все Q мгновенно становятся 0, игнорируя CLK».

## Фаза 3.5: Структурный синтез RTL-блоков (Datapath Library)

* [x] **Структурный AIG-конструктор (без TruthTable-blowup):** Новый модуль [nand_optimizer/structural.py](nand_optimizer/structural.py). Класс `StructuralModule(name, inputs)` накапливает AIG через примитивы `not1 / and2 / nand2 / or2 / nor2 / xor2 / xnor2 / mux2(sel, a, b) / and_tree / or_tree / const0 / const1`; все примитивы возвращают AIG-литералы и прозрачно используют structural hashing (вторичный вызов с теми же аргументами возвращает тот же литерал без новых узлов). Метод `add_output(name, lit)` объявляет выходы; `finalize(script, verbose)` прогоняет заданный синтез-скрипт через `run_script()` (по умолчанию `'rewrite; fraig; balance'`), затем через `aig_to_gates()`, и возвращает `OptimizeResult`-совместимый объект с заполненными `.builder.gates`, `.outputs[name].out_wire`, `.aig`, `.out_lits`, `.total_nand` и `._MockTT`-заглушкой в `.truth_table` (достаточной для `export_circ`). Прецедент: 8-битный реверсивный счётчик (26 входов, $2^{26} \approx 67$М минтермов — таблица физически нереализуема) строится за миллисекунды и даёт 248 NAND.
* [x] **Параметризованная библиотека datapath-блоков:** Новый модуль [nand_optimizer/datapath.py](nand_optimizer/datapath.py) поверх `structural.py`. Генераторы: `half_adder(m, a, b)→(sum, carry)`, `full_adder(m, a, b, cin)→(sum, cout)`, `ripple_adder(m, a, b, cin)→(sums, cout)` (LSB-first, каскад `full_adder`); `eq_comparator(m, a, b)` (поразрядный XNOR + AND-дерево); `zero_detect(m, a)` (AND всех NOT); `ones_detect(m, a)` (AND-дерево); `ripple_up_carry(m, q, cin=TRUE)→List[Lit]` (ladder $C_{i+1}=C_i \land Q_i$, длина n+1) и `ripple_down_borrow(m, q, bin=TRUE)→List[Lit]` ($B_{i+1}=B_i \land \lnot Q_i$); `mux2_bus(m, sel, a, b)` (поразрядный mux); `priority_encoder(m, xs)→(valid, encoded)` (grant-каскад от старшего входа к младшему, ⌈log₂ n⌉ выходов); `jk_excitation(m, q, q_next)→(J, K)` (T-fill: `J=K=Q XOR Q_next`, structural hashing сливает J и K в один узел). Все блоки работают на литералах, общие подграфы автоматически разделяются.
* [x] **Встроенный пример: универсальный реверсивный счётчик на JK-триггерах (ЛР №2):** [nand_optimizer/examples/jk_counter.py](nand_optimizer/examples/jk_counter.py) с фабрикой `universal_reversible_counter(bits=8)`. Синтезирует комбинационный JK-возбуждающий конус (J_i, K_i) для N-битного счётчика: LOAD·D_i / count-up via ripple_up_carry / UP-rollover at LIMIT → reset to 0 / count-down via ripple_down_borrow / DOWN-wrap at Q=0 → load LIMIT. Ключевое исправление vs. наивного TODO-варианта: `Count_enable = ~Rollover & (UP | ~Z)` подавляет carry/borrow-toggle в момент терминального события, предотвращая интерференцию цепочки переносов с логикой обёртки. Результат (8-bit): 248 NAND, 26 входов, 16 выходов; 4-bit: 123 NAND. Функция `run_jkcounter_regression(bits, verbose)`: 256 шагов × 4 сценария (count-up, count-down, parallel-load, rollover-at-LIMIT) cycle-accurate симуляция против Python reference; все 1024 шага проходят. CLI: `python -m nand_optimizer jkcounter --bits 8 --circ counter8.circ`; добавлен аргумент `--bits N` в `__main__.py`; `.circ` через `export_circ` с `_MockTT`-заглушкой (valid XML, 1076 компонентов для 8-bit).

* [x] **care_rounds_internal как флаг `dc -C N`:** Реализовано в [nand_optimizer/script.py](nand_optimizer/script.py) (коммит `f232c33`). Добавлен парсер флага `-C N` в команду `dc`, который маппируется в параметр `rounds` функции `dc_optimize()`. Использование: `dc -C 2` для 2 раундов итеративной care-propagation, либо комбинированно `dc -C 3 --odc -K 5`. * [x] **DCO V2 — инструментация, adaptive-W и исправление exhaustive-enum:** Пополнены счётчики в [`last_dc_stats()`](nand_optimizer/dont_care.py): `n_sim_retries`, `final_sim_W`, `n_inputs`. Добавлены параметры `adaptive_sim` (default False) и `max_sim_W=16384`: при срабатывании safety-net-miter'а пасс ретраит себя на удвоенном W (кап 16384, MAX_RETRIES=2). Core-path выделен в `_dc_optimize_once`, возвращающий `(aig, out, reverted)`. Найден и исправлен off-by-one в exhaustive-PI-enumeration: условие `(1 << n_inputs) > n_sim_patterns` падало в random-sampling при W == 2^n_inputs (n=7 → W=128 покрывал только ~63% пространства, revert на ctrl). Теперь `if n_inputs <= SIM_ENUM_THRESHOLD:` всегда использует exhaustive. Прогон EPFL subset (10 бенчмарков) с W=128/W=1024, `adaptive_sim ∈ {on, off}` показал: (1) на `ctrl` (7 inputs) исправление exhaustive даёт −30.5% ANDs (174→121) без ретраев; (2) на `router`/`priority`/`i2c`/`sin` (>14 inputs) все ревёрты стабильны под ретраями до 16384 — это не sim-undersampling, а теоретический зазор V2 admissibility под reconvergent fanout; (3) adaptive-W жжёт время без выигрыша (sin: 86s→416s на все-реверт'е). Поэтому adaptive_sim оставлен off-by-default как отладочный флаг, а настоящая soundness-gap вынесена в отдельный активный пункт TODO. Логи: `/tmp/dc_v2_epfl_probe.{log,json}`.

* [x] **Multi-Armed Bandit контроллер скрипта синтеза (`ScriptBandit`):** Реализовано в [`nand_optimizer/script.py`](nand_optimizer/script.py) как класс `ScriptBandit` с двумя стратегиями: **UCB1** (`arg max μᵢ + c·√(2·ln t / nᵢ)`) и **Thompson Sampling** (Beta-prior, reward clipping в [0,1]). Arms по умолчанию — single-command скрипты `balance`, `rewrite`, `rewrite -z`, `fraig`, `dc` (настраиваемы через `arms=[...]`). На каждом шаге: `select()` возвращает index ручки, снаружи прогоняется соответствующая команда через `run_script`, `update()` получает reward = `(n_before - n_after) / n_before` по числу AND-узлов. `run_bandit(aig, out_lits, horizon, strategy)` — standalone-оркестратор, возвращает trace выбранных команд. Интегрировано в `optimize()` через параметры `bandit_horizon=N`, `bandit_strategy={'ucb1','thompson'}`; в CLI — `--bandit HORIZON`, `--bandit-strategy`. `--bandit` оверрайдит `--script`. FlowTune-стиль подход: без нейросети, накопленная статистика прокачивает вероятности выбора без pre-training.

* [x] **Don't-Care V3 — устранение soundness gap под reconvergent fanout:** Корень проблемы V2 — не undersampling, а admissibility coverage: для схем с 60–147 PI exhaustive-проверка `W=16384` всё равно ревертит, значит sim-based admissibility в принципе недостаточна. Реализованы три механизма:

  1. **`odc_mode='z3-exact'`** ([dont_care.py](nand_optimizer/dont_care.py)): для каждого template и каждой resub-операции выполняется точная Z3-проверка `UNSAT(ODC_v AND T(cut) ≠ old_v)`, где `ODC_v` вычисляется через `z3.substitute` (кэшируется по узлу — O(n_nodes × n_POs) один раз, затем O(SAT) на check). CLI: `dc --odc --odc-mode z3-exact`. Результаты на EPFL:

     | benchmark | n_ands | reverts legacy | reverts z3-exact | area delta |
     |-----------|--------|----------------|-----------------|------------|
     | router    | 257    | 1              | **0** ✓         | −2.3%      |
     | priority  | 978    | 1              | **0** ✓         | −21.9%     |
     | i2c       | 1342   | 1              | **0** ✓         | −0.1%      |
     | sin       | 5416   | 1              | 1 (>3000 AND → legacy fallback) | 0% |

     Скорость: 4–27 s на benchmark vs. 0.2–2 s legacy (10–15× overhead из-за Z3 per-template). `sin` превышает порог `n_ands ≤ 3000` и автоматически падает в legacy.

  2. **`odc_mode='window'`** ([dont_care.py: `_propagate_care_sim_window`](nand_optimizer/dont_care.py)): forward-flip per-node симуляция в K-уровневом fanout-окне вместо глобального care propagation. Меньше reconvergence-рисков, полезен для малых схем как консервативная альтернатива. CLI: `dc --odc --odc-mode window --window-depth K`. На фикстуре router (depth=1, W=32768) reverts=0; на полных EPFL-схемах coverage-проблема admissibility check сохраняется при глубоких путях.

  3. **Topology-aware 1-gate resub window** ([dont_care.py: `_scan_resub_1gate`](nand_optimizer/dont_care.py)): при переполнении пула (`len(items) > max_window`) вместо FIFO-хвоста выбираются `max_window` узлов, чей уровень в `new_aig` ближайший к уровню целевого узла. Уровни заполняются инкрементально в `_sync_sig_new` при добавлении новых AND-узлов. Инструментация: счётчики `n_resub_1gate_examined` (итераций в hot-loop) и `n_resub_1gate_dropped_by_window` (пар, пропущенных из-за усечения) добавлены в `last_dc_stats()`.

  **Regression:** минимальный revert-воспроизводящий cone из `router` outport[1] извлечён delta-debug с constant-substitution: [tests/fixtures/router_outport1_minimal.aig](tests/fixtures/router_outport1_minimal.aig) — 14 AND / 15 inputs / 1 output. Тест-файл [tests/test_dc_odc_soundness.py](tests/test_dc_odc_soundness.py) содержит 3 инварианта: (1) `dc` без `--odc` не ревертит фикстуру; (2) `dc --odc --odc-mode z3-exact` не ревертит; (3) `dc --odc` legacy ревертит фикстуру (live-сигнал, что legacy gap ещё не залатан для этого входа).

  **Прочие изменения:** удалён устаревший параметр `tfo_cap` из `dc_optimize()` (был CLI-совместимым заглушкой ещё с V1, игнорировался в V2); поля `sta` и `switching` добавлены в `OptimizeResult` для будущей интеграции STA/switching-aware optimization.

## Фаза 7 (продолжение)

* [x] **ATPG (Automatic Test Pattern Generation):** Новый модуль [nand_optimizer/atpg.py](nand_optimizer/atpg.py). SAT-based stuck-at fault detection и test-pattern generation. Функции:
  - `enumerate_stuck_at_faults(aig, output_lits)` → `List[Fault]` — перечисление всех stuck-at-0 и stuck-at-1 неисправностей на каждом узле
  - `compute_fault_coverage(aig, output_lits)` → `float` — метрика покрытия на случайных входных паттернах (симуляция fault-free vs. faulty)
  - `sat_test_pattern(aig, output_lits, fault)` → `Optional[Dict[input, bool]]` — SAT-запрос miter'а (original vs. faulty), возвращает вектор, различающий две версии, или None если неразличимы
  
  Формальное основание: каждый stuck-at запрос — двухуровневый miter (исходный AIG плюс fault-injected копия) с XOR-OR деревом выходов; Z3-solver ищет удовлетворяющий assignment. Используется для анализа тестопригодности.

* [x] **Switching Activity Estimation:** Новый модуль [nand_optimizer/switching.py](nand_optimizer/switching.py). Вероятностное распространение переключательной активности по AIG:
  - `estimate_switching_activity(aig, output_lits, input_prob=0.5)` → `Dict[node_id, float]` — вероятность переключения каждого узла `p_sw(v) = p(v) * (1 - p(v))`, где `p(v)` — signal probability на узле
  - Правила for AND: `p_out = p_a && p_b` (независимые входы); для NOT: `p_out = 1 - p_in`
  - Опция `correlation_aware=True` учитывает reconvergent paths через cofactor-анализ (вычислительно дороже, но точнее)
  
  Основа для power-aware оптимизации: узлы с высокой `p_sw` — кандидаты на минимизацию глитчей, buffer-insertion, перестройку дерева для снижения ёмкости на критических узлах. Интегрируется с `balance.py` и будущим power-aware rewrite.
